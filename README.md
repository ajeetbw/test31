# Sprint Call Data Record (CDR) Processor
CDR specifies charging functionality and charging management for cellular service providers, handling CDR files that contain CDR records. Each CDR record contains descriptions of charging events such as voice calls and SMS messages with all relevant data.

## Project Description
The Sprint Call Data Record (CDR) Processor is designed to process CDR files generated by the network, which are then transferred to the network operator’s Billing domain (BD) for the purpose of subscriber billing and inter-operator accounting. Offline charging is a process where charging information for network resource usage is collected concurrently with that resource usage. The charging information is then passed through a chain of logical charging functions, resulting in the generation of CDR files.

### CDR File Format
The CDR files contain a header section followed by a variable sized CDR data section. The CDR data section contains zero or more concatenated CDR records. Each CDR record is encoded on a single line and composed of the following fields separated by `|`:
* MSISDN: unique number identifying a subscription in a mobile network (maximum length of 7 digits)
* Operator Brand Name: a string (maximum 64 characters)
* Operator MMC/MNC: mobile country code and mobile network code (maximum 6 digits)
* Call type: one of the following:
	+ MOC: outgoing voice call
	+ MTC: incoming voice call
	+ SMS-MO: outgoing message
	+ SMS-MT: incoming message
	+ GPRS: internet
* Duration: in seconds
* Download: MB downloaded (if type is GPRS)
* Upload: MB uploaded (if type is GPRS)
* Third party MSISDN: corresponding third party in this call (empty for GPRS)
* Third party operator MMC/MNC: corresponding third party’s mobile operator

### Processing Functions
The CDR processor fulfills the following functions:
1. Customer Billing
2. Inter Operator Settlement

## Tech Stack
The project utilizes the following technologies:
* **Programming Languages:** Java, Python
* **Frameworks:** Spring Boot, Flask
* **Libraries:** Apache Commons, pandas
* **Databases:** MySQL, PostgreSQL

## Installation and Startup
To install and start the CDR processor, follow these steps:
1. Clone the repository using `git clone`
2. Install the required dependencies using `pip install -r requirements.txt` (for Python) or `mvn install` (for Java)
3. Configure the database connection settings in the `application.properties` file (for Java) or `config.py` file (for Python)
4. Start the application using `java -jar target/cdr-processor.jar` (for Java) or `python app.py` (for Python)

## Basic Usage
The CDR processor provides the following API endpoints:
* `POST /process-cdr`: processes a CDR file and returns the processed data
* `GET /customer-billing`: retrieves customer billing information
* `GET /inter-operator-settlement`: retrieves inter-operator settlement information

Example usage:
```bash
curl -X POST -H "Content-Type: application/json" -d '{"cdrFile": "path/to/cdr/file"}' http://localhost:8080/process-cdr
```
